Writing Good Variable Names in A Clean Code

Using Good and sensible variable names is very important in a software developer's carrer.
It makes you to write good code and be a world-class developer in the run.

Following things should be taken care of while assigning names to your variables

1. MUST USE INTENTION-REVEALING NAMES
i.e the variable name should be self-explanatory about what the variable does

It must tell :-
a. the reason of existense of the variable
b. what does the variable do
c. how is the variable used

Why?
because in a team there are multiple people contributing to the same project at same time
and with some random variable name one will have a hard time reading/understanding our code
even we ourselves may forget,
what is that 'z' variable meant for?

Good variable names help to make keep your code alive forever i.e it is useful always

Choosing names that reveal intent can make it much easier to understand and change the code.

---------------------------------------------------------------------------------------------------

2. Different variables should be named such that one can distinguish in between them
since they are different so needless to say they must appear different too.

---------------------------------------------------------------------------------------------------

3. Must use easily-pronounceable names

because codes are usually discussed it a team and you need to use tha name frequently
but if you can't utter the name easily, you won't be able to dicuss it with ease.

---------------------------------------------------------------------------------------------------
4. Must use searchable names so that if you are facing a bug in a application which you coded a year back then

you should be able to go and search on basis of general instinct
because there are high chances that you forget

about the name which you gave to the variable
which was holding details of the richest person in the town
was it 'x' or 'richestInTown'.
and yeah did I mention that some other random developer might also sit down to debug that application.
how will he search?
by ringing you up in the midnight to ask the name of that variable.........??
Choice is yours

-------------------------------------------------------------------------------------------------------
5. Avoid encoding

Write names such that even a dumb person is able to understand your code
Leave it to the final layer to encode your code and take care of the security vulnerabilities
-------------------------------------------------------------------------------------------------------
6. Avoid Mental Mappings

The variables name should be such that you don't have to remember that
why this piece of variable is used here
also use domain-specific names so that the names aren't generic and you don't have to struggle to get a clear
idea of the meaning of the variable name

--------------------------------------------------------------------------------------------------------

7. Class Names and Method names

Use nouns -> for classes and objects namings
use verbs -> for methods namings

---------------------------------------------------------------------------------------------------------

8. Use Memorable names

we should use universal names for variable names.
Not slangs or words which only a few people understand

----------------------------------------------------------------------------------------------------------

9. Pick one word per concept

Either delete or remove -> to delete a object
choose one word for deleting a object and use that same word through-out the code
Be consistent with the word you choose

Consistency is the key here

------------------------------------------------------------------------------------------------

10. Don't Pun

We shouldn't use the same word for two differnt purposes

-----------------------------------------------------------------------------------------------

11. Use Solution and Problem Domain Names

While writing variable if some technical term and be used which is obvious to the
programmers world to understand , then use it
Don't dive deep to be domain specific.

Don't be cold-blooded for your own community Programmers

------------------------------------------------------------------------------------------------

12. Add meaningful contexts

Add some prefixes to variable name so that it gives more clear idea of the context
either use a class to encapsulate them

------------------------------------------------------------------------------------------------

13. If you find any variable name in somebody else's code which can be renamed to be a better version then change it.
Don't be scared,
You are doing good Man!
Go for it.....!!!
